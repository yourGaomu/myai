# Paper Reader 使用指南

## 快速开始

### 1. PDF 论文研读

**步骤：**
1. 上传 PDF 文件到对话中
2. 说："请帮我读一下这篇论文" 或 "总结这篇PDF"

**示例对话：**
```
用户：[上传 paper.pdf]
用户：请帮我读一下这篇论文

助手：好的，我来帮你研读这篇论文...
[生成完整的研读报告]
```

### 2. arXiv 论文

**支持格式：**
- 完整 URL：`https://arxiv.org/abs/2301.07001`
- PDF URL：`https://arxiv.org/pdf/2301.07001.pdf`
- arXiv ID：`arXiv:2301.07001` 或 `2301.07001`

**示例：**
```
用户：帮我总结 arXiv:2301.07001

助手：正在获取 arXiv 论文信息...
[生成研读报告]
```

### 3. DOI 论文

**格式：** `10.xxxx/xxxxx`

**示例：**
```
用户：分析这篇论文：10.1038/s41586-019-1234-5

助手：正在解析 DOI...
[生成研读报告]
```

## 输出模式

### 完整模式（默认）

生成包含以下内容的详细报告：
- 基本信息
- 研究问题
- 主要贡献
- 方法论分析
- 实验结果
- 批判性思考
- 研读笔记
- 相关资源

**触发方式：**
- "请帮我读一下这篇论文"
- "详细分析这篇论文"
- "论文研读"

### 简要模式

生成精简版报告，聚焦核心内容：
- 一句话总结
- 三大贡献
- 核心方法
- 关键结果

**触发方式：**
- "简要总结这篇论文"
- "快速摘要"
- "这篇论文的核心贡献是什么"

### 专题模式

聚焦特定方面的分析：

**方法论分析：**
- "这篇论文的方法有什么创新"
- "分析这篇论文的技术路线"

**实验分析：**
- "这篇论文的实验设计如何"
- "分析实验结果"

**批判性思考：**
- "这篇论文有什么局限性"
- "这篇论文的优缺点"

## 高级功能

### 保存研读笔记

**自动保存：**
在 `config.json` 中设置 `"save_notes": true`

**手动保存：**
```
用户：保存这篇论文的研读笔记

助手：已保存到 ~/.openclaw/workspace/papers/[论文标题].md
```

### 对比分析

```
用户：对比这两篇论文 [上传两个PDF]

助手：[生成对比报告]
```

### 生成引用

```
用户：给我这篇论文的引用格式

助手：
BibTeX:
@article{...}

APA:
作者. (年份). 标题. 期刊.
```

## 配置选项

编辑 `config.json` 自定义行为：

```json
{
  "output_language": "zh-CN",        // 输出语言：zh-CN / en
  "detail_level": "comprehensive",   // 详细程度：brief / standard / comprehensive
  "include_references": true,        // 是否分析参考文献
  "save_notes": true,                // 是否自动保存笔记
  "notes_path": "~/.openclaw/workspace/papers/",  // 笔记保存路径
  "arxiv_mirror": "https://arxiv.org",  // arXiv 镜像站点
  "max_pdf_size_mb": 50,             // 最大 PDF 文件大小
  "enable_ocr": false                // 是否启用 OCR（扫描版 PDF）
}
```

## 常见问题

### Q: PDF 解析失败怎么办？

**可能原因：**
1. PDF 是扫描版（需要 OCR）
2. PDF 加密或损坏
3. PDF 文件过大

**解决方法：**
1. 尝试使用 OCR 工具预处理
2. 检查 PDF 是否可正常打开
3. 压缩或分割 PDF

### Q: arXiv 论文获取失败？

**可能原因：**
1. 网络连接问题
2. arXiv ID 格式错误
3. arXiv 服务暂时不可用

**解决方法：**
1. 检查网络连接
2. 确认 arXiv ID 格式正确
3. 稍后重试或直接上传 PDF

### Q: 如何处理中文论文？

目前优先支持英文论文。中文论文可以：
1. 直接上传 PDF 尝试解析
2. 提供论文链接
3. 手动提供关键信息

### Q: 生成的报告太长/太短？

调整 `config.json` 中的 `detail_level`：
- `brief`：简要版
- `standard`：标准版
- `comprehensive`：详细版

## 最佳实践

### 1. 明确需求

告诉助手你想要什么：
- "我需要了解这篇论文的核心方法"
- "帮我评估这篇论文的实验设计"
- "这篇论文对我的研究有什么启发"

### 2. 提供上下文

说明你的背景和目的：
- "我是做 NLP 的，想了解这篇 CV 论文"
- "我在复现这篇论文，需要注意什么"
- "我在写综述，需要总结这篇论文"

### 3. 追问细节

对感兴趣的部分深入讨论：
- "这个方法的具体实现细节是什么"
- "为什么选择这个评估指标"
- "这个结果和预期一致吗"

## 技巧提示

### 💡 快速筛选论文

使用简要模式快速了解论文价值：
```
用户：简要总结这篇论文

助手：[生成快速摘要]
用户：看起来不错，请详细分析
```

### 💡 批量研读

上传多个 PDF，逐个分析：
```
用户：我有3篇论文要读 [上传3个PDF]
用户：先读第一篇
助手：[研读第一篇]
用户：继续第二篇
助手：[研读第二篇]
```

### 💡 专题深入

针对特定问题深入分析：
```
用户：这篇论文的实验设计有什么问题？
用户：如何改进这个方法？
用户：这个方法能应用到我的场景吗？
```
