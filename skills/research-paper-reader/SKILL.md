# Research Paper Reader - 论文研读总结助手

## 功能概述

这是一个专门用于学术论文研读和总结的技能，能够：

- 📄 **多格式支持**：PDF文件、arXiv链接、DOI、论文URL
- 🎯 **智能提取**：自动识别论文结构（摘要、方法、结果、结论）
- 📊 **结构化总结**：生成清晰的论文总结报告
- 🔍 **深度分析**：方法论评估、实验设计分析
- 💡 **批判性思考**：识别论文的优缺点、局限性和未来方向
- 📝 **笔记生成**：生成可保存的研读笔记
- 📚 **文献库管理**：自动索引已读论文，支持检索和关联
- 🔗 **上下文记忆**：追踪阅读历史，生成研究进展报告
- 🎯 **相关推荐**：基于主题、关键词、作者推荐相关论文

## 核心功能

### 1. 论文研读

支持多种输入格式：

```
请帮我读一下这篇论文 [上传PDF]
总结 arXiv:2301.07001
分析这篇论文：10.1038/s41586-019-1234-5
```

### 2. 文献库管理

自动管理已读论文：

```
查看我的文献库
列出最近阅读的论文
搜索关于[主题]的论文
```

### 3. 上下文总结

追踪研究进展：

```
总结我最近的研究进展
生成阅读报告
查看我的研究洞察
```

### 4. 相关论文推荐

基于已有论文推荐：

```
推荐与这篇论文相关的论文
找出类似方法的论文
```

## 工作流程

### Step 1: 论文获取与解析

根据输入类型自动选择获取方式：

| 输入类型 | 识别规则 | 获取方式 |
|---------|---------|---------|
| PDF 文件 | 用户上传的附件 | 直接读取文件内容 |
| arXiv ID | `arXiv:\d{4}\.\d{4,5}` 或 `arxiv.org/abs/` | 通过 arXiv API 获取元数据 + PDF |
| DOI | `10.\d{4,}/[^\s]+` | 通过 DOI 解析获取论文信息 |
| URL | `https?://.*\.pdf` | 下载 PDF 文件 |

### Step 2: 内容提取与分析

1. **全文文本提取**：提取所有文本内容
2. **结构识别**：识别章节标题
3. **关键信息提取**：标题、作者、机构、关键词
4. **深度分析**：研究问题、方法、结果、贡献

### Step 3: 文献库索引

自动将论文添加到文献库：

```json
{
  "id": "paper_xxxxx",
  "title": "论文标题",
  "authors": ["作者1", "作者2"],
  "year": "2026",
  "keywords": ["关键词1", "关键词2"],
  "topics": ["主题1", "主题2"],
  "summary": "一句话总结",
  "contributions": ["贡献1", "贡献2"],
  "added_date": "2026-04-12T19:37:00"
}
```

### Step 4: 上下文更新

更新阅读历史和研究进展：

- 记录阅读时间
- 追踪研究主题
- 积累研究洞察
- 生成进展报告

### Step 5: 相关论文推荐

基于以下维度推荐相关论文：

- 📌 **关键词匹配**：相同或相似关键词
- 📌 **主题关联**：相同研究领域
- 📌 **作者关联**：相同作者的其他论文
- 📌 **引用关系**：相互引用的论文

## 文献库管理

### 数据结构

```
~/.openclaw/workspace/papers/
├── library/
│   ├── index.json              # 主索引
│   ├── papers/                 # 论文详情
│   └── notes/                  # 研读笔记
├── by-topic/                   # 按主题分类
├── by-author/                  # 按作者分类
├── by-date/                    # 按日期分类
├── reading_history.json        # 阅读历史
├── current_session.json        # 当前会话
└── research_insights.md        # 研究洞察
```

**index.json**:
```json
{
  "papers": {
    "paper_xxxxx": {
      "id": "paper_xxxxx",

      "authors": [...],
      "keywords": [...],
      "topics": [...],
      "summary": "...",
      "contributions": [...],
      "related_papers": [...]
    }
  },
  "topics": {
    "machine-learning": ["paper_xxxxx", ...]
  },
  "authors": {
    "John Smith": ["paper_xxxxx", ...]
  },
  "keywords": {
    "neural network": ["paper_xxxxx", ...]
  }

```

### 检索功能

支持多种检索方式：

1. **按标题搜索**：
   ```
   搜索标题包含"attention"的论文
   ```

2. **按主题搜索**：
   ```
   查看关于"自然语言处理"的论文
   ```

3. **按作者搜索**：
   ```
   找出"Yann LeCun"的论文
   ```

4. **按关键词搜索**：
   ```
   搜索关键词"transformer"
   ```

5. **组合搜索**：
   ```
   找2023年关于"大语言模型"的论文
   ```

## 配置选项

可在 `config.json` 中配置：

```json
{
  "output_language": "zh-CN",
  "detail_level": "comprehensive",
  "include_references": true,
  "save_notes": true,
  "notes_path": "~/.openclaw/workspace/papers/",
  "auto_index": true,
  "max_related_papers": 10,
  "context_days": 30,
  "enable_recommendations": true
}
```

### 配置说明

- `output_language`：输出语言
- `detail_level`：详细程度
- `include_references`：是否包含参考文献分析
- `save_notes`：是否自动保存研读笔记
- `notes_path`：笔记保存路径
- `auto_index`：是否自动索引到文献库
- `max_related_papers`：最多推荐相关论文数
- `context_days`：上下文统计天数
- `enable_recommendations`：是否启用相关推荐

## 技术实现

### 脚本文件

- `fetch_arxiv.py`：获取 arXiv 论文
- `parse_pdf.py`：解析 PDF 文件
- `library_manager.py`：文献库管理
- `context_manager.py`：上下文管理

### 数据存储

- **文献库索引**：JSON格式，支持快速检索
- **阅读历史**：按会话组织，支持时间追踪
- **研读笔记**：Markdown格式，易于阅读和编辑
- **研究洞察**：Markdown格式，支持导出

## 注意事项

1. **版权问题**：仅用于个人学习和研究
2. **数据安全**：文献库数据存储在本地
3. **隐私保护**：不上传论文内容到云端
4. **性能优化**：大型文献库可能需要优化索引
