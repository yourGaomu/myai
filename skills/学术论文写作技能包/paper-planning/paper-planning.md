# 学术论文大纲规划 (Paper Planning)

## 触发条件

用户提到以下任意场景时使用本 Skill：
- "写大纲"、"论文规划"、"规划论文"、"paper plan"
- "论文结构"、"章节规划"、"outline"
- "帮我规划一篇论文"

## 概述

从研究方向+实验结论出发，生成一篇完整论文的结构化大纲。核心产出是**论断-证据矩阵**和**逐节规划**，确保论文的每一条论断都有实验支撑，每一节都有明确目的。

## 输入

接受以下任一形式：
1. **NARRATIVE_REPORT.md**（最佳）— 结构化研究叙述文件路径
2. **研究方向 + 实验结论** — 口头或文字描述
3. **PAPER_PLAN.md** 草稿 — 已有大纲需要审查优化

## 操作步骤

### Step 1: 提取论断与证据

构建 **Claims-Evidence Matrix**（论断-证据矩阵）：

```markdown
| # | Claim | Evidence | Status | Section |
|---|-------|----------|--------|---------|
| C1 | 我们的方法在X任务上优于SOTA | 实验A, 指标B=85.3% vs 83.1% | ✅ 已验证 | §4.1 |
| C2 | 消融实验证明模块Y的关键作用 | 实验C, 去掉Y后下降5.2% | ✅ 已验证 | §4.3 |
| C3 | 方法具有计算效率优势 | 实验D, FLOPs对比 | ⚠️ 部分验证 | §4.2 |
```

**一句话贡献**：提炼论文最核心的一句话贡献陈述。这是整篇论文的"灵魂"，后续所有写作围绕它展开。

### Step 2: 确定论文类型和结构

根据研究性质选择论文结构：

| 论文类型 | 推荐结构 | 页数分配 |
|---------|---------|---------|
| **实证论文** | Intro → Related Work → Method → Experiments → Analysis → Conclusion | 1.5 + 1 + 1.5 + 3 + 1 + 0.5 |
| **理论+实验** | 可拆为7-8节，理论分为估计+优化两段或设定+分析 | 灵活分配 |
| **方法论论文** | Intro → Related Work → Method → Experiments → Ablation → Conclusion | 1.5 + 1 + 2 + 2.5 + 1 + 0.5 |

### Step 3: 逐节规划

对每节指定：核心内容、关键词、预估长度、引文需求、图表位置。

**Abstract 五部分结构**：
1. 我们做到了什么（具体贡献）
2. 为什么重要/困难
3. 如何做到的（方法一句话）
4. 证据
5. 最强的量化结果

**Introduction 关键要素**：
- 开头 hook（1-2句问题动机）
- 明确差距（"However, ..."）
- 方法概述（在读者迷失前给出全貌）
- 2-4个可证伪的贡献列表
- 主要结果预览
- Hero Figure 描述

### Step 4: 图表规划

```markdown
| ID | 类型 | 描述 | 数据来源 | 优先级 |
|----|------|------|----------|--------|
| Fig 1 | Hero/Architecture | 系统概述+对比示意 | manual | 🔴 HIGH |
| Fig 2 | 折线图 | 训练曲线对比 | exp_A.json | 🔴 HIGH |
| Table 1 | 对比表 | 主要结果 vs baselines | main_results.json | 🔴 HIGH |
| Fig 3 | 消融柱状图 | 模块贡献分析 | ablation.json | 🟡 MED |
```

### Step 5: 引文脚手架

每个 section 列出需要的引文类别，格式如：
```
§1 Introduction
  - 领域背景综述: [AuthorYYYY] 类型
  - SOTA方法: 具体方法名 + 第一作者 + 年份
  - 差距动机: 需要支撑"当前方法未解决X"的引用

§2 Related Work
  - 子方向A: [AuthorYYYY]...
  - 子方向B: [AuthorYYYY]...
```

### Step 6: 独立审稿

使用内置审稿能力对完整大纲进行独立审阅（审阅过程在本地完成，大纲内容不外传），评估：
1. **逻辑流** — 故事是否自然推进？
2. **论断-证据对齐** — 每个论断都有支撑？
3. **缺失的实验或分析** — 审稿人可能要求什么？
4. **与先前工作的定位** — 差异是否清晰？
5. **页面预算可行性** — 8页内能否装下？

## 输出

- **PAPER_PLAN.md** — 完整结构化大纲文件
- **GAP_REPORT.md**（如使用 style-ref）— 与目标论文的风格差距分析

## 与其他 Skill 的关系

- 输出 PAPER_PLAN.md 可被 `paper-latex-writing` 直接使用
- 图表规划可交付给 `paper-figures` 生成实际图表
- 完成后建议进入 `paper-latex-writing` 开始撰写

## 注意事项

- 不要跳过 Claims-Evidence Matrix — 它是全流程的质量锚
- 规划时注意页面预算，避免规划过多内容
- 独立审稿应使用全新内置审稿上下文线程（本地处理，不外传），确保独立性
